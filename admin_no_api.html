1 初回セットアップ（一度だけ）

手順操作内容
①ニュースポータル管理DB を開き「拡張機能」→「Apps Script」
②既存コードを全削除し、news_portal_no_api.gs の内容を貼り付けて保存
③関数選択で「createForm」を実行 → Googleフォームが自動作成される
④関数選択で「setupTriggers」を実行 → 権限を許可
⑤「showFormUrl」を実行してログに表示されたURLをブックマーク
⑥スプレッドシートの共有設定：「リンクを知っている全員」→「閲覧者」に変更
③④の実行時に「権限の確認」ダイアログが出たら「詳細」→「（安全ではないページ）に移動」→「許可」を選択してください。

2 毎日の記事入力

ブックマークしたフォームURLを開いて入力します。1回の送信で1記事です。複数記事は「別の回答を送信」で続けて入力できます。

項目入力内容例
対象日付記事の対象日2026/04/23
セクションカテゴリ選択optics（光学・光アプリ）
タグ短いラベル光学、フォトニクス
情報ソース・媒体名出典（省略可）日経新聞、Reuters
ニュースタイトル記事タイトル○○社が△△を発表
詳細説明2〜5文の説明本文を要約して入力
インサイト・考察1〜2文の考察（省略可）業界への影響など
ソースURL元記事のURL（省略可）https://...
💡 送信後すぐにサイトに反映されます（index.html をリロードして確認）。

3 PDF・音声のアップロード

記事入力とは独立して、PDFと音声ファイルを自動スキャンします。

news-portal-data/
  └── 2026-04-23/ ← 日付フォルダ（YYYY-MM-DD形式）
      ├── tech.pdf      ← 技術系レポートPDF
      ├── politics.pdf  ← 政治情勢レポートPDF
      └── audio.mp3     ← 音声解説（10分以内に自動反映）
ファイル名は固定です：tech.pdf / politics.pdf / audio.mp3（または .m4a / .wav）
☁️ Driveフォルダを開く

+ 管理用関数

関数名用途
showFormUrl()フォームURLをログに表示（ブックマーク忘れた場合）
checkStatus()各シートのデータ件数とフォーム状態を確認
scanNow()Driveメディアスキャンを今すぐ手動実行
deleteArticlesByDate('2026-04-22')指定日付の記事を全削除（入力ミスのリセット）
createForm()フォームを再作成（フォームを誤削除した場合）
